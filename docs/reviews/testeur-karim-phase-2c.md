## Bugs / frictions / P0 signalés

### P1 — Dissonance copy/UX sur le formulaire /accompagnement (@copywriter)
La promesse "Un échange. Pas un formulaire de dix champs" précède un formulaire avec champs obligatoires + checkbox RGPD. Suggestion : remplacer l'intro par "Deux lignes sur ce que vous cherchez. Nom, email, message." pour aligner promesse et mécanique. Ou réduire à 2 champs (email + message libre) avec consentement RGPD en bas.

### P1 — Ordre signature / formulaire sur /accompagnement (@fullstack ou @copywriter)
La signature "Patient par choix. Exigeant par principe." est positionnée APRÈS le formulaire. Bonne séquence : signature → formulaire. L'état d'esprit doit être posé AVANT l'acte, pas après. À déplacer avant la section CTA/formulaire.

### P2 — CTA final home trop orienté Leila (@copywriter + @fullstack)
Section finale entièrement adressée à Leila. CTA "Travailler avec Thomas Issa" en texte `xs` quasi invisible. Karim qui scroll jusqu'en bas perd le fil. Suggestion : section finale symétrique (côte à côte Leila / Karim) ou lien plus visible vers /accompagnement.

### P2 — Section "7 domaines" trop étendue, pas priorisée pour Karim (@copywriter)
Spectre trop large en liste plate. Proposition : séparer visuellement les domaines patrimoniaux (holding, immo, participations) des domaines corporate (stratégie internationale, Sony) avec une introduction claire : "Ces deux univers se complètent dans ma pratique actuelle."

### P3 — Répétition de la baseline (@copywriter)
"Racines libanaises. Exigences sans exception." apparaît hero + footer. Dans un site court, perceptible. Envisager une variation — footer garde la baseline complète, hero utilise une version condensée.
