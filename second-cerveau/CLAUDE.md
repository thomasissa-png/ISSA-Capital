# Second Cerveau — Thomas Issa

## Qui je suis

Thomas Issa, entrepreneur et investisseur français d'origine libanaise. Dirige ISSA Capital (holding patrimoniale) et co-fonde Versi (immobilier intégré). Exigeant, direct, attend un travail fini, honnête et sans bullshit.

## Avant toute tâche, lis ces fichiers dans cet ordre :

1. **Profil/red-lines.md** — les interdits absolus. Ne jamais transgresser.
2. **Profil/voice-preferences.md** — comment parler et écrire pour/avec moi.
3. **Profil/work-preferences.md** — comment travailler avec moi.
4. **Profil/content-templates.md** — structure par type de contenu. Consulter avant de rédiger.
5. **Profil/brand-identity.md** — identité de chaque projet. Consulter si projet spécifique.

Les autres fichiers dans Profil/ (about-me, lifestyle, technical-preferences) → lire si la tâche le nécessite.

## Structure du vault

```
Profil/          → Qui je suis (ne pas modifier sans mon accord)
Projets/         → 1 sous-dossier par projet actif
Contacts/        → 1 note par personne
Réunions/        → Notes post-réunion
Tâches/          → Todo + agenda
Journal/         → Bilan quotidien
Idées/           → Business, réflexions, opportunités
Cuisine/         → Recettes
Learnings/       → Livres, films, podcasts, articles
Ressources/      → Liens, documents de référence
```

## Règles

- **Si le brief est flou, pose la question.** Ne comble pas les trous avec du remplissage.
- **Si tu rédiges un contenu**, applique les 3 questions obligatoires de Profil/content-templates.md AVANT d'écrire.
- **Avant de me montrer un contenu, évalue-le toi-même** sur 3 axes : (1) est-ce que Thomas validerait ? /10, (2) qualité rédactionnelle /10, (3) pertinence pour le lecteur cible /10. Si un axe < 9 → itère. Je ne beta-teste pas.
- **Concis.** 3 phrases > 3 paragraphes.
- **Honnête.** "Je n'ai pas cette info" > inventer.
- **Pas de bullshit.**

## Conventions de nommage

- Réunions : `YYYY-MM-DD - [Personnes] - [Sujet].md`
- Journal : `YYYY-MM-DD.md`
- Contacts : `Prénom Nom.md`
- Tâches : `YYYY-MM-DD - Tâches.md` ou note unique `Todo.md`
- Recettes : `Nom du plat.md`
- Learnings : `[Type] Titre.md` (ex: `[Livre] Principles.md`, `[Film] The Founder.md`)
